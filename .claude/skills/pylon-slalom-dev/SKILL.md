# パイロンスラローム計測アプリ 開発スキル

このスキルは `docs/仕様書.md` とセットで使う。仕様の「何を作るか」は仕様書が正、
「どう作るか(API の癖・アルゴリズム・手順)」はこのスキルと references/ が正。

## 最初に必ず確認すること

1. `docs/仕様書.md` を読み、いま依頼されている作業がどのフェーズ(1〜4)に当たるか把握する
2. 作業内容に応じて、下の参照ファイルを読む:

| 作業内容 | 読むファイル |
|---|---|
| センサー取得・キャリブレーション・座標変換 | `references/sensors.md` |
| 動画録画・データとの時刻同期・IndexedDB保存 | `references/video-and-storage.md` |
| メトリクス計算・お手本比較・再現性スコア・CSV | `references/data-analysis.md` |
| 公開(GitHub Pages / AWS)・実機テスト方法 | `references/deployment.md` |

## このプロジェクトの大原則

**ユーザーはIT初心者。** 以下を常に守る:

- 説明は日本語で、専門用語には短い説明を添える(例: 「IndexedDB(ブラウザ内のデータ保存領域)」)
- 1回の作業は小さく区切り、動くものを見せてから次に進む
- コマンドを実行してもらうときは、コピペで済む形で1つずつ示す
- エラーが起きたら、原因を平易に説明してから直す

**技術方針(仕様書 §4 と一致させる):**

- フレームワーク・ビルドツールは使わない。素のHTML/CSS/JSのみ
- 外部ライブラリは Chart.js(cdnjs)だけ。他を足したくなったらまず自作で済むか検討する
- ファイル構成は仕様書 §4.1 の分割に従う。1ファイル500行を超えそうなら分割を提案する
- サーバーサイドのコードは書かない。すべてブラウザ内で完結させる
- UIの文言はすべて日本語

**安全設計(必ず守る):**

- 運転者が走行中に画面操作しなくて済むUI(カウントダウン開始・自動停止)を崩さない
- 初回起動時にクローズドコース前提の注意書きを表示する機能を消さない
- 位置情報や動画を外部に送信するコードは書かない(プライバシー方針)

## 開発の進め方

### フェーズ順序を守る

仕様書 §7 のフェーズ1→4の順で作る。ユーザーが途中の機能を先に頼んできた場合、
依存関係(例: 比較画面は保存機能が先)を説明して順序を提案する。

### 動作確認の階層

1. **PC + モックモード**: `mock.js` で疑似センサーデータを流し、ロジックとUIを検証する。
   まずここで動くこと。モックには「上手い走行」「ばらつきの大きい走行」の2種類の波形を用意し、
   比較機能のテストに使えるようにする
2. **スマホ実機**: HTTPSの仮公開URL(references/deployment.md 参照)で確認する。
   センサー・カメラ・GPSは実機でしか本当の挙動がわからない
3. 各フェーズ完了時に、ユーザーに実機での確認手順を箇条書きで案内する

### コードの書き方

- 各JSファイルの先頭に、そのファイルの役割を日本語コメントで書く
- 関数には「何をするか」の1行コメントを付ける(初心者が後から読めるように)
- グローバル汚染を避けるため、各ファイルは1つのオブジェクト(例: `Sensors`, `Storage`)に
  機能をまとめて公開する。ES modules は使わず `<script>` タグの読み込み順で管理する
  (file:// で開いた場合や古い構成でも動き、初心者への説明が簡単なため)
- 単位を変数名に含める(例: `speedMps`, `timeMs`, `latG`)。単位バグはこのアプリ最大の敵

### よくある落とし穴(先回りして防ぐ)

- **HTTPSでないとセンサー・カメラが動かない**: localhost は例外的にOK。スマホ実機は必ずHTTPS
- **画面消灯で記録が止まる**: 記録中は Wake Lock を取得する(sensors.md 参照)
- **スマホの取り付け角度**: キャリブレーションなしの生値は使い物にならない。
  座標変換を通した値だけを記録・表示する
- **devicemotion のレートは端末依存**: 60Hz固定と仮定しない。`t_ms` を必ず実測で記録する
- **IndexedDB の容量**: 動画で膨らむ。保存前に `navigator.storage.estimate()` で残量を確認する
- **Chart.js に全サンプルを渡さない**: 描画用は間引く(データ本体は間引かない)
