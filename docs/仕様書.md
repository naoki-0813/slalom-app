- 版数: 1.0
- 作成日: 2026-07-21
- 目的: Claude Code でこのアプリを開発するための要件定義・技術仕様

---

## 1. 概要

### 1.1 何を作るか

スマートフォンを車に固定し、約100mの区間に設置したパイロン(スラローム)をすり抜ける走行を、スマートフォンのセンサーで記録・分析する **ブラウザ上で動くWebアプリ**。

できること:

1. 走行中のセンサーデータ(加速度・角速度・GPS)を記録する
2. 走行中の前方映像をスマホカメラで録画する
3. 記録したデータを **CSVファイルとして保存** できる
4. 録画した映像を **動画ファイルとして保存** できる
5. 走行結果をグラフで閲覧できる
6. **熟練者(お手本)の走行と自分の走行を重ねて比較** できる
7. **自分の複数回の走行を重ねて再現性(ばらつき)をチェック** できる

### 1.2 使い方のイメージ

1. スマホを車のダッシュボード等に前方が映る向きで固定する
2. ブラウザでアプリのURLを開く
3. 「キャリブレーション」で停車状態を数秒記録(スマホの傾き補正)
4. 「記録開始」をタップ → カウントダウン後に記録スタート(運転者は運転に集中)
5. スラローム走行する(記録は自動または同乗者が停止)
6. 結果画面でグラフ・動画を確認し、CSV・動画を保存
7. 熟練者の走行データと重ねて比較、または自分の走行同士を比較

### 1.3 対象環境

| 項目 | 内容 |
|---|---|
| 対象スマホ | Android + Chrome(最新版) |
| 必須条件 | HTTPSでの配信(カメラ・センサーAPIの制約) |
| 通信 | 記録・分析はすべて端末内で完結(オフラインでも動作するのが理想) |
| サーバー | 不要(静的ファイルのみで動く構成にする) |
| PC(開発用) | Chrome。センサーがないためモック(疑似データ)モードを用意する |

### 1.4 前提・安全上の注意(アプリ内にも表示すること)

- クローズドコース(私有地・練習場)での使用を前提とする
- 運転者は走行中にスマホを操作しない。開始はカウントダウン方式、停止は自動または同乗者が行う
- 本アプリは運転練習の補助であり、計測値の正確性を保証するものではない

## 2. 画面仕様

シングルページアプリ(SPA)とし、以下の5画面を切り替える。UIは日本語。ボタンは走行前でも押しやすいよう大きくする。

### 2.1 ホーム画面

- 「記録開始」「走行一覧」「比較」「設定」への大きなボタン
- 初回はキャリブレーション未実施の警告を表示

### 2.2 記録画面

- カメラのプレビュー表示(前方映像)
- 「キャリブレーション」ボタン: 停車状態で3秒間の重力ベクトルを記録し、スマホ座標系→車両座標系の変換を確定する
- 「記録開始」ボタン: 5秒カウントダウン(音+画面表示)後に記録開始
- 記録中の表示: 経過時間、現在の横G(大きな数字)、記録中インジケータ
- 停止方法(両方実装):
  - 手動停止ボタン(同乗者用)
  - 自動停止: 「発進(速度>5km/h)を検知してから、停止(速度<2km/hが3秒継続)で自動終了」
- 記録中は画面消灯を防止(Wake Lock)

### 2.3 結果画面(1本の走行の詳細)

- 走行メタ情報の編集: 走行名、日時(自動)、ドライバー名、「お手本(熟練者)」フラグのON/OFF
- グラフ表示(下記メトリクス参照):
  - 横G(車両左右方向)の時系列
  - ヨーレート(旋回角速度)の時系列
  - 速度の時系列
- サマリー数値: 区間タイム、最大横G、平均速度、スムーズネススコア
- 動画プレイヤー: 録画映像を再生。グラフをタップするとその時刻に動画がシークする(動画とデータは同じ時刻基準で同期)
- 保存ボタン:
  - 「CSVを保存」→ 端末のダウンロードフォルダへ
  - 「動画を保存」→ .webm ファイルとしてダウンロード

### 2.4 比較画面

- 走行一覧から2〜5本を選択して重ね合わせ表示
- モードA「お手本と比較」: お手本フラグ付き走行を基準(太線)にし、自分の走行を重ねる
- モードB「再現性チェック」: 自分の走行同士を重ね、ばらつきを見る
- 表示内容:
  - 横G・ヨーレート・速度の重ね合わせグラフ(横軸は「区間の進行率0〜100%」に正規化)
  - 比較スコア表: 基準との差(RMSE)、タイム差、最大横G差、スムーズネス差
  - 再現性スコア: 選択した走行間の一致度(0〜100点)
- 講評コメント: スコアに応じた簡単なアドバイス文を自動表示(例: 「切り返しのタイミングがお手本より遅れています」)

### 2.5 走行一覧・設定画面

- 走行一覧: 保存済み走行のリスト(名前/日時/タイム/お手本マーク)、詳細表示・削除・CSV/動画の再ダウンロード
- 設定: 記録レート表示、自動停止のON/OFF、動画画質(720p/480p)、データ全削除、モックモード切替(開発用)

## 3. 機能仕様

### 3.1 センサー記録

| データ | 取得API | レート | 用途 |
|---|---|---|---|
| 加速度(重力除く) | `devicemotion` の `acceleration` | 端末依存 約60Hz | 横G・前後G |
| 加速度(重力込み) | `devicemotion` の `accelerationIncludingGravity` | 同上 | キャリブレーション |
| 角速度 | `devicemotion` の `rotationRate` | 同上 | ヨーレート |
| 端末姿勢 | `deviceorientation` | 約60Hz | 補助 |
| 位置・速度 | `geolocation.watchPosition`(高精度モード) | 約1Hz | 速度・軌跡・自動停止 |

- 各サンプルには記録開始からの経過ミリ秒 `t_ms` を付与する(`performance.now()` 基準)
- 動画も同じ基準時刻で開始時刻を記録し、データと同期させる
- **座標変換(重要)**: スマホの取り付け角度は毎回異なるため、キャリブレーションで得た重力方向から回転行列を作り、センサー値を車両座標系(前後X・左右Y・上下Z)に変換して記録する

### 3.2 動画記録

- `getUserMedia`(背面カメラ、720p、30fps目安)+ `MediaRecorder`(webm/VP8またはVP9)
- 100m区間の走行(数十秒)を想定。1本あたり概ね10〜50MB
- 記録終了後、Blobを走行データと紐付けてIndexedDBに保存

### 3.3 データ保存(端末内)

- 保存先: **IndexedDB**(ブラウザ内データベース。サーバー不要・オフラインOK)
- 保存単位は「走行(run)」: メタ情報 + センサー配列 + 動画Blob
- 端末の空き容量に注意し、一覧画面に使用容量を表示。古い動画のみ削除する機能を付ける

### 3.4 CSV仕様

ファイル名: `run_{走行名}_{YYYYMMDD-HHMMSS}.csv`(UTF-8、ヘッダー行あり)

```csv
t_ms,ax_veh,ay_veh,az_veh,gx,gy,gz,yaw_rate,speed_mps,lat,lon,gps_accuracy_m
0,0.12,-0.03,0.01,0.5,-0.2,1.1,1.1,0.0,35.68123,139.76712,4.5
16,0.15,-0.10,0.02,0.4,-0.1,2.3,2.3,0.0,,,
```

- `ax_veh/ay_veh/az_veh`: 車両座標系の加速度 [m/s²](Xが前後、Yが左右、Zが上下)
- `gx/gy/gz`: 車両座標系の角速度 [deg/s]、`yaw_rate` = 車両Z軸まわり
- GPSは約1Hzのため、GPS列は更新があった行のみ値を入れ、他は空欄
- 先頭にコメント行(`#` 始まり)で走行メタ情報(走行名・日時・キャリブレーション値・アプリバージョン)を出力

### 3.5 分析メトリクス

| メトリクス | 定義 |
|---|---|
| 区間タイム | 発進検知(速度>5km/h)〜停止検知までの秒数 |
| 最大横G | \|ay_veh\| の最大値(1G=9.81m/s²で表示) |
| スムーズネススコア | 横Gの時間微分(ジャーク)のRMSが小さいほど高得点(0〜100に正規化) |
| 切り返し回数 | ヨーレートの符号反転回数(パイロン数の目安と一致するか確認できる) |
| 再現性スコア | 走行間の横G波形の一致度(進行率で正規化後のRMSE・相関係数から算出、0〜100) |
| お手本との差 | 進行率で正規化した横G・ヨーレート波形のRMSE、タイム差、切り返しタイミングのずれ |

正規化の方法: 各走行の発進〜停止を0〜100%として同じ点数(例: 200点)にリサンプリングしてから比較する。詳細アルゴリズムはスキルの `references/data-analysis.md` を参照。

## 4. 技術構成

| 項目 | 選定 | 理由 |
|---|---|---|
| フレームワーク | なし(素のHTML/CSS/JavaScript) | 初心者が読める・ビルド不要・不具合の切り分けが簡単 |
| グラフ | Chart.js(cdnjsから読み込み) | 実績が多く軽量 |
| データ保存 | IndexedDB(薄いラッパー関数を自作) | サーバー不要・動画Blobも保存可 |
| ファイル構成 | `index.html` / `css/style.css` / `js/`(機能別に数ファイル) | 1ファイル巨大化を避ける |
| ビルド・サーバー | 不要。開発時は `python3 -m http.server` 等 | シンプル |
| PWA | 後回し(フェーズ4の任意項目) | まず動くものを優先 |

### 4.1 JSファイル分割(目安)

```
js/
├── app.js        # 画面切り替え・全体制御
├── sensors.js    # センサー取得・キャリブレーション・座標変換
├── recorder.js   # 動画録画・記録セッション管理(開始/自動停止)
├── storage.js    # IndexedDB保存・CSV生成・ダウンロード
├── analysis.js   # メトリクス計算・正規化・比較スコア
├── charts.js     # グラフ描画・動画シーク連携
└── mock.js       # PC開発用の疑似センサーデータ
```

## 5. ホスティング(公開方法)

スマホの実機テストにはHTTPSのURLが必要。以下のいずれかを選ぶ(未決定。どちらでも動く作りにする)。

| 方式 | 費用 | 特徴 |
|---|---|---|
| A. GitHub Pages | 完全無料 | GitHubアカウントがあれば最短。AWSは使わない |
| B. AWS Amplify Hosting | 無料枠内でほぼ0円 | AWSに慣れる目的にも合う。Gitと連携して自動デプロイ |
| C. AWS S3 + CloudFront | 無料枠内でほぼ0円 | 構成の学習になるが手順は最多。S3単体はHTTPSにならない点に注意 |

- 開発中のスマホ実機確認は、まずGitHub PagesかAmplifyで仮公開するのが簡単(localhostはスマホから直接開けないため)
- 手順の詳細はスキルの `references/deployment.md` を参照

## 6. 非機能要件

- プライバシー: データはすべて端末内に保存し、外部送信しない(その旨をアプリ内に明記)
- 性能: 60Hz×60秒 ≒ 3,600サンプル程度。配列処理で十分軽い。グラフは間引き描画してよい
- 画面: 縦持ち・横持ち両対応(記録画面は横持ち推奨の案内を表示)
- エラー処理: センサー・カメラ・GPSの権限拒否時は、日本語でわかりやすい案内を表示する
- ブラウザ対応: Android Chrome最新版のみ保証(iOS対応は将来課題として設計だけ壊さない)

## 8. 用語集(初心者向け)

- **横G**: カーブで体が横に押される力。スラロームの上手さが最もよく表れる値
- **ヨーレート**: 車が向きを変える速さ(旋回の角速度)
- **キャリブレーション**: スマホの取り付け角度のずれを補正するための基準取り
- **IndexedDB**: ブラウザの中にあるデータ保存領域。サーバーがなくてもデータを残せる
- **HTTPS**: 暗号化された通信方式。カメラやセンサーを使うWebアプリでは必須
